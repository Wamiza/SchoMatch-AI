# 5-Minute Demo Plan for Kaggle Judges

This plan is optimized to showcase the maximum value of SchoMatch-AI within a tight 5-minute evaluation window.

## Preparation (Before recording)

1. Ensure `.env` is configured with a valid `GOOGLE_API_KEY`.
2. Run `make seed` to ensure the database has the rich sample opportunities.
3. Start the application using `docker-compose up -d`.
4. Open `http://localhost:5173` in a clean browser window.

## Demo Flow

### 0:00 - 1:00: The Problem & Solution (Home Page)
- **Action**: Start on the landing page.
- **Talk Track**: "Welcome to SchoMatch-AI. Students today face a massive hurdle: finding relevant academic opportunities requires manually filtering thousands of websites with complex eligibility rules. SchoMatch solves this using a multi-agent system powered by Google ADK and Gemini."
- **Highlight**: Point out the premium UI, responsive design, and the "How It Works" 3-step process.

### 1:00 - 2:00: The Multi-Agent Pipeline (Discover Page)
- **Action**: Click "Start Discovering". Fill the form with a specific profile (e.g., Computer Science, GPA 3.4, Master's, interests in AI, seeking Scholarships and Research).
- **Talk Track**: "Here, the student inputs their profile. When I hit submit, we aren't just doing a database lookup. We trigger an ADK `SequentialAgent` pipeline."
- **Action**: Hit Submit.
- **Highlight**: Focus the screen on the **Agent Pipeline Status** component. As it updates from Profile Analysis -> Discovery -> Eligibility -> Career Advisor -> Deadlines, explain what each agent is doing under the hood (e.g., "The Eligibility agent is using Gemini 2.5 Pro to reason about the strict requirements vs the student's profile").

### 2:00 - 3:30: The Results (Dashboard Page)
- **Action**: Once the pipeline completes, the app transitions to the Dashboard.
- **Talk Track**: "Here are the AI-curated matches. Notice the Match Score ring."
- **Action**: Expand a specific opportunity card (e.g., "Google Summer of Code" or "DAAD Scholarship").
- **Highlight**: Show the **Eligibility Summary**, the **Recommendation Reason** (generated by the AI), and specifically point out the **Missing Requirements** (e.g., if the GPA was too low, the AI flags it).

### 3:30 - 4:30: Career Advice & Action Plan
- **Action**: Scroll to the left sidebar showing the Career Advisor Notes and Action Plan.
- **Talk Track**: "This isn't just a search engine. Our Career Advisor agent generated this personalized action plan based on the student's specific gaps. Since the student had a 3.4 GPA, it advised focusing on raising it to a 3.5 for elite scholarships, and outlined exactly which application to start first based on the Deadline Tracker agent's schedule."

### 4:30 - 5:00: Architecture & Wrap-up
- **Action**: Briefly show the code structure (specifically `agent_runner.py` showing the ADK setup).
- **Talk Track**: "Under the hood, this uses FastAPI, React, and 4 MCP servers (Search, Knowledge, File, External APIs). It's fully dockerized, secure, and ready to scale. Thank you."
